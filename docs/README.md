## Document Purpose

- Master Specification: the consolidated product and system definition.
- Requirements Assessment: accepted, unclear, deferred, and changed requirements.
- Architecture Decisions: the decisions that should shape code.
- Backend Use Cases And Tests: module use cases and test targets.
- Roadmap: phased implementation order.
- Validation Script Requirements: architecture enforcement rules.
- Project Log: long-term session memory.
- Next Actions: short active task list.
